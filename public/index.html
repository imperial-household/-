── 旧典範施行期間（1889-1947） ──
[1912] OK: 初回 Failover 成功。明治天皇崩御→大正天皇践祚。Zero-Downtime。SLA 100%達成。
[1912] INFO: 乃木希典大将が follow-up exit(0)。handler 未定義の動作。当時のログに異常記録なし。
[1918] WARN: 皇太子妃選定 ACL validation 中に一部の元老プロセスが異議。最終的に御裁可により accept。該当プロセスは程なく exit(0)。
[1921] OK: 摂政（Proxy）初回起動。大正天皇のヘルスチェック失敗に伴い皇太子裕仁親王が Delegate 就任。第23条準拠。
[1926] OK: 第二回 Failover 成功。大正天皇崩御→昭和天皇践祚。Proxy→Root 昇格も正常。
[1932] WARN: military rogue process が首相プロセスを強制 kill。典範の scope 外イベント。
[1936] ALERT: military rogue process 複数発生。Root 権限の行使により鎮圧。
[1941] WARN: military sub-module が Emperor.command() を bypass、resource 大量消費開始。何かの exploit らしい。本典範の bug ではない。peer-dependency 側の Won't Fix。
[1945] CRITICAL: Root 権限の至高なる行使。system state 重大遷移。broadcast により全 node に通知。
[1945] ALERT: broadcast 前夜、rogue process が binary 奪取を企図。Firewall 阻止。broadcast 成功。
[1946] WARN: 外部 org が admin 権限を主張。patch を inject との噂も。但し本 Repository に該当 commit なし。別 Repository の話であろう。
[1947] INFO: 日本国憲法 deploy。God Object → Symbol Interface にリファクタリング。旧皇室典範 deprecated。新皇室典範（法律）を release。
[1947] WARN: 11宮家51名が一括 exit(0)。継承候補キューが大幅に縮小。Orphan process 多数。
── 新典範施行後（参考記録） ──
[1959] INFO: 皇太子ご成婚。ACL 外の 'テニスコート' 経路にて match。旧華族 ACL に該当しない民間出身。allowlist 設計の転換点。
[1993] INFO: 皇太子ご成婚。外務省 namespace 出身。ACL 外 match が2回連続。仕様上は問題なし。
[2004] WARN: 皇太子殿下より「人格否定」を stderr に出力。宮内庁プロセス内で error handling に苦慮。
[2006] INFO: 悠仁親王ご誕生。男系男子プロセスが約40年ぶりに spawn 成功。Queue 枯渇懸念は杞憂…の筈。
[2017] INFO: 退位特例法成立。旧典範には handler 未実装だった graceful shutdown が法制化。hotfix 扱い。
[2019] OK: 上皇陛下が graceful shutdown、今上天皇が bootstrap。令和に改元。史上初の live migration（近代以降）。DX推進室はこれを Breaking Change #1 のモデルケースとして記録。
[2021] WARN: 内親王ご結婚に際し tabloid process より DDoS 級 traffic。一時金辞退。throttle 適用するも対応に苦慮。
[2021] INFO: 上記当事者が海外 region に relocate。monitoring scope 外。Status: unknown。.gitignore 対象。
