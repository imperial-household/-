【はじめに】

本システムは、明治22年（1889年）に制定された旧皇室典範を
TypeScript で実装したレガシーシステムです。

宮内庁DX推進室では、このレガシーコードを令和の皇室典範に
モダナイズするための破壊的変更（Breaking Change）を
本ブランチ（main）で準備しています。
master ブランチには旧典範の原文（帝国時代のコード）が保存されています。

なお、本システムはIT用語による注釈付き解説を含む
風刺的教育コンテンツとしての側面を持ちます。

【本システムの概要】

一、天皇（CurrentEmperor）プロセス管理
　　日本国憲法第1条に定める「日本国の象徴」としての
　　天皇プロセスの状態遷移（在位・退位・崩御）を管理します。
　　天皇は国政に関する権能を有しません（憲法第4条）。

二、皇位継承（Succession）の記録
　　皇位継承順序の管理と、継承イベント発生時の
　　プロセス切替手順を定義しています。
　　令和元年には退位特例法に基づく生前退位が実現し、
　　本システムにも退位ハンドラが実装されました。

三、DX推進室対応Bot（koho-response-bot）
　　外部からの Issue や Pull Request に対し、
　　DX推進室が移行作業中である旨をご案内し、
　　クローズする自動応答システムです。

四、DX推進室モダナイゼーションCI（shoryobu-archive-ci）
　　DX推進室が管理するモダナイゼーションのための
　　継続的インテグレーション環境です。
　　型チェック、構造検証、ビルド確認を自動実行し、
　　Breaking Change の導入前に整合性を検証します。

【文字符号化方式について】

本文書は UTF-8 で符号化されています。
旧字体から新字体への変換は完了していますが、
一部の固有名詞・法令用語に歴史的表記が残る場合があります。

【本システムの運用記録（制定より現在まで）】

  旧皇室典範は明治22年の制定以来、昭和22年の廃止まで
  約58年間にわたり施行されました。
  以下はその間の主要な運用記録です。

  ◆ 初回 Failover（大正元年・1912年）
  　 明治天皇崩御に伴い、皇太子嘉仁親王へ皇位継承。
  　 践祚即位の手続きにより Zero-Downtime で切替完了。
  　 ※ 乃木希典大将が殉死（follow-up exit(0)）。
  　 　 仕様外の動作ですが、当時の社会的文脈では
  　 　 異常とは見なされませんでした。

  ◆ 摂政（Proxy）初回起動（大正10年・1921年）
  　 大正天皇のご病状に伴い、皇太子裕仁親王が
  　 摂政に就任。第23条に基づく Delegate 起動です。
  　 Proxy Pattern の初の実運用となりました。

  ◆ 第二回 Failover（昭和元年・1926年）
  　 大正天皇崩御に伴い、摂政から天皇へ昇格。
  　 ダウンタイムなしで切替完了。

  ◆ 即位礼 Deploy（昭和3年・1928年）
  　 即位の礼及び大嘗祭を京都にて執行。
  　 第11条の定めに基づく儀式的 Deploy です。

  ◆ 戦後の大規模リファクタリング（昭和22年・1947年）
  　 日本国憲法の施行に伴い、旧皇室典範は廃止。
  　 新たな皇室典範が国会の議決により法律として制定されました。
  　 天皇の位置づけは「統治権の総攬者」から「象徴」へ変更。
  　 God Object → Symbol Interface への移行が完了。
  　 この破壊的変更こそ本プロジェクトの元ネタです。
  　 ※ 11宮家51名が皇籍を離脱。Queue の大幅縮小が発生しましたが、
  　 　 新制度の下で運用が継続されています。

  ◆ 令和の退位イベント（平成31年・2019年）
  　 天皇の退位等に関する皇室典範特例法に基づき、
  　 上皇陛下が退位、今上天皇が即位されました。
  　 旧典範には存在しなかった「退位」状態の追加は、
  　 本システムにおける初の Breaking Change 実績です。
  　 DX推進室はこれをモデルケースとして今後の改修を推進中。

【使用技術】

  言語:     TypeScript（型付き写本語）
  実行環境: Node.js
  自動化:   GitHub Actions
  公開基盤: GitHub Pages

【権限体系（現行制度に基づく整理）】

  天皇（Emperor）     : Symbol Interface / 国事行為のみ
  皇嗣（Successor）   : 待機系統 / ReadOnly
  国会（Diet）         : 立法権 / Write権限あり（皇室典範の改正権者）
  国民（Citizens）     : 主権者 / 閲覧・参政権あり
  外国人（Foreigners） : 閲覧可能（CORS: Access-Control-Allow-Origin: *）

  ※ 旧典範では天皇に「Root権限」が設定されていましたが、
  　 現行憲法下では Symbol Interface として再定義されています。
  　 国政に関する権能を有しない点が設計上の特徴です。

【よくある質問（FAQ）】

  Q: 天皇のご体調に関する Health Check ログは閲覧できますか？
  A: 宮内庁の広報を通じて公式発表されるものが全てです。
  　 プライバシーに関わる情報は公開対象外です。

  Q: 婚姻に関する承認プロセスの詳細は？
  A: 皇室会議の議を経る手続きが皇室典範に定められています。
  　 歴史的な経緯を含む詳細はDX推進室の資料をご参照ください。

  Q: 皇族数の減少による継承候補の不足は問題では？
  A: 現在、有識者会議等で議論が行われている重要な課題です。
  　 本システムの設計上は Queue の縮小に対する
  　 Contingency Plan が十分とは言えず、
  　 将来的な制度改正が必要とされています。

  Q: 旧皇族の復帰という議論がありますが？
  A: 現行皇室典範第15条（旧典範第34条相当）により、
  　 皇族の身分を離れた者は復帰できない規定ですが、
  　 法改正により変更可能です。国会での議論事項です。

  Q: 天皇の配偶者選定に関して国民の意見は反映されますか？
  A: 皇室のご意向と皇室会議の手続きに基づきます。
  　 国民の関心は報道を通じて知ることができます。
